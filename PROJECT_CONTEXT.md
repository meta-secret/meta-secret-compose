## What this is

Kotlin Multiplatform (KMM) mobile app for **Android** (Jetpack Compose) and **iOS** (UIKit + minimal Swift proxy). It integrates a **prebuilt** Rust native library (`meta-secret-lib`) over FFI for cryptography and secret sharing.

- **Android:** UniFFI-generated Kotlin (JNA) + `libmetasecret_mobile.so` (see `System.setProperty` for `uniffi.component.mobile_uniffi.libraryOverride`)
- **iOS:** `metasecret-mobile.a` + generated Swift under `iosApp/UniffiGenerated` (`mobile_uniffiFFI` module map; bridging header remains for Foundation-only)

## Relation to `meta-secret-core`

Rust sources and the UniFFI/mobile build live in the separate **`meta-secret-core`** repository. This repo **consumes** the native artifacts; it does not edit Rust. Cross-repo changes follow **core first**, then integration here (see [CLAUDE.md](CLAUDE.md)).

## Build commands

```bash
./gradlew build
./gradlew installDebug
./gradlew :composeApp:detekt
./gradlew test
./gradlew test --tests "com.example.MyTestClass"
```

iOS builds use **Xcode** directly. The Kotlin framework is generated by Gradle and embedded into the Xcode project.

## Platform validation

- **iOS:** treat **physical iPhone** as the reference environment. Do not assume the iOS Simulator is available or relevant.
- **Android:** emulator or physical device.
- **Build success does not imply launch success.**

## Runtime debugging (summary)

- For **iOS** runtime issues, use a manual device-log workflow. Ask for crash message, Xcode console, device log snippet, screenshot if needed.
- Do not declare a runtime issue resolved until the user confirms verification or explicitly defers it.
- Separate **build** failures from **runtime** failures in diagnosis.
